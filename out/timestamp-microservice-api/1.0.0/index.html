timestamp-microservice-api 1.0.0

  • User Story: I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2016).
  • User Story: If it does, it returns both the Unix timestamp and the natural language form of that date.
  • User Story: If it does not contain a date or Unix timestamp, it returns null for those properties.

API Usage

/{date}

Accepts a {date}, either in the form of a UNIX timestamp or a human readable string, and returns a JSON object with both the UNIX timestamp and human readable string.

For example:

https://timestamp-microservice-logan.herokuapp.com/May 23 1970, https://timestamp-microservice-logan.herokuapp.com/May%2023%201970, https://timestamp-microservice-logan.herokuapp.com/12268800,

will all return

{"unix":12268800,"natural":"May 23 1970"}
